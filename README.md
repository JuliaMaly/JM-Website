## Опис продукту

Цей проєкт є повноцінним **fullstack веб-застосунком**, який складається з клієнтської (frontend) та серверної (backend) частин.
---
### Архітектура

Застосунок побудований за клієнт-серверною архітектурою:

- **Frontend** — реалізований на React (Vite)
- **Backend** — реалізований на Node.js з використанням Express
- Взаємодія відбувається через REST API

---

### Основний функціонал

Застосунок дозволяє:

- виконувати авторизацію користувача
- переглядати список товарів (Products)
- створювати нові товари
- видаляти товари
- переглядати список замовлень (Orders)
- видаляти замовлення
- навігувати між сторінками через інтерфейс

---
## Як запустити проєкт

### 1. Клонувати репозиторій

```
git clone https://github.com/JuliaMaly/JM-Website.git
```

---

### 2. Запуск backend

```
cd lab2
npm install
node server.js
```

Backend працює на:

```
http://localhost:3000
```

---

### 3. Запуск frontend

(у новому терміналі)

```
cd frontend-lab3
npm install
npm run dev
```

Frontend працює на:

```
http://localhost:5173
```

---

### Важливо

- backend і frontend мають працювати одночасно
- дані очищаються після перезапуску сервера

---

### Авторизація

Реалізована базова система авторизації:

- форма логіну (email + password)
- перевірка даних через API (`/auth/login`)
- отримання токена
- збереження токена в `localStorage`
- збереження стану користувача через Context API

Для тестування використовується:

```
email: admin@gmail.com
password: 123456
```

### Управління станом (State Management)

Для управління станом використано:

- **React Context API**
- зберігання токена та даних користувача
- синхронізація з `localStorage`

---

### Робота з API

Frontend взаємодіє з backend через HTTP-запити:

- `POST /auth/login` — авторизація
- `GET /products` — отримання товарів
- `POST /products` — створення товару
- `DELETE /products/:id` — видалення товару
- `GET /orders` — отримання замовлень
- `DELETE /orders/:id` — видалення замовлення

---

### Особливості реалізації

- використання сучасного React (hooks)
- SPA (Single Page Application)
- розділення логіки на компоненти та сервіси
- чиста структура проєкту
- взаємодія frontend ↔ backend

---

### Обмеження

- дані зберігаються в пам’яті (in-memory), без бази даних
- після перезапуску сервера всі дані очищаються
- авторизація є демонстраційною (без реального JWT)

---

### Висновок

Проєкт демонструє:

- розробку REST API
- створення клієнтського застосунку на React
- інтеграцію frontend і backend
- роботу з авторизацією та станом
- базові принципи побудови веб-застосунків
